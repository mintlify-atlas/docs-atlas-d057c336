## How it works

Mirror unifies server and client into a single project. Instead of `MonoBehaviour`, you extend `NetworkBehaviour` and use Mirror's attributes to declare what gets synchronized and which methods are remote calls:

```csharp Player.cs
public class Player : NetworkBehaviour
{
    // Automatically synced from server to all clients
    [SyncVar] public int health = 100;

    // Client calls this → runs on server
    [Command]
    void CmdUseItem(int slot) { /* validated on server */ }

    // Server calls this → runs on all clients
    [ClientRpc]
    void RpcRespawn() { /* play respawn animation */ }

    void Update()
    {
        if (isServer) ServerUpdate();
        if (isClient) ClientUpdate();
    }
}
```

## Get started

<Steps>
  <Step title="Install Mirror">
    Download Mirror from the [Unity Asset Store](https://assetstore.unity.com/packages/tools/network/mirror-129321) or install via the Package Manager.
  </Step>
  <Step title="Add a NetworkManager">
    Create a new GameObject and attach the **Network Manager** component, then add your transport of choice.
  </Step>
  <Step title="Create a NetworkBehaviour">
    Replace `MonoBehaviour` with `NetworkBehaviour` on your player script and add a `NetworkIdentity` component to the prefab.
  </Step>
  <Step title="Add to NetworkManager">
    Drag your player prefab into the NetworkManager's **Player Prefab** field and press Play.
  </Step>
</Steps>

<Note>
Mirror supports Unity 2019, 2020, 2021, 2022 LTS, and Unity 6. Check the [installation guide](/getting-started/installation) for version-specific notes.
</Note>
